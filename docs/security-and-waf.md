# 安全与 WAF

## 定位

内置 WAF 位于应用 HTTP 路由之前，用于降低随机不存在 URL、重复动态请求、搜索、XML-RPC 和附件下载对服务资源的消耗。它不是反向代理、CDN 或专业边缘 WAF 的替代品；生产环境仍应设置连接数、请求体、TLS 和网络层限制。

WAF 有总开关，各策略也可独立启停。配置从 `gb_options` 读取并在进程内缓存 5 秒，因此保存后短时间内可能仍有旧配置请求正在执行。

## 请求处理顺序

进程启动层先根据 `data/config.json` / `--allow-cidr` 检查 TCP 直接来源地址。不在允许网段内的请求直接返回 `403`，不会进入 WAF、插件或后台管理员白名单；该设置用于限制整个服务的网络暴露面，不属于可在后台修改的 WAF 规则。

通过启动层网段检查后，WAF 内部主要顺序为：

1. 加载 WAF 配置并安装安全响应头 writer。
2. 总开关关闭时直接进入应用，但安全响应头仍会应用。
3. 已登录管理员访问后台路径时绕过 WAF 限流/封禁；非 GET/HEAD 完成后使公开缓存失效。
4. 解析客户端 IP并检查现有封禁。
5. 执行动态、静态或附件资源分类限流。
6. 执行搜索和 XML-RPC 专项限流。
7. 插件路由直接进入应用，避免公开 URL 索引误判。
8. 对适用请求查询公开 URL 索引；不存在时统计无效路径并可能封禁。
9. 对可缓存公开请求读取/写入响应缓存。
10. 对附件下载统计次数，达到阈值时封禁。
11. 进入实际路由；写请求完成后使公开索引和缓存失效。

理解顺序很重要：命中页面缓存前仍会经过 IP 封禁和限流；插件路由可绕过 URL 存在性检查，但不会绕过前面的频率限制。反向代理来源网段必须同时通过启动层 CIDR，后台的转发头信任设置才有机会生效。

## 管理员白名单边界

只有同时满足以下条件才绕过 WAF：

- 请求路径是 `/admin` 或 `/admin/...`。
- 当前会话有效。
- 用户角色至少为 `administrator`。

这不是按来源 IP 的永久白名单。管理员访问前台仍按公开请求处理，编辑和普通用户后台请求也不会获得同样的全局绕过。认证仍使用正常会话签名与用户 `authCode`，不能靠请求头声明管理员身份。

## 公开 URL 索引

URL 索引把实际存在的公开路径维护在内存集合中，随机不存在 URL可在进入内容数据库详情查询前快速返回 404。

索引包括首页、Feed、兼容接口、文章和页面固定链接、分类/标签及其 Feed、作者和归档等公开路径，并限制最大条目数。索引按 TTL 刷新，写操作会主动失效。

以下请求不会简单按静态索引判断：带身份 Cookie的页面、搜索、归档参数、预览、Trackback、后台和静态资源等需要特殊处理的路径。已启用插件注册的路由也单独识别。

新增核心公开路由时必须更新索引生成或排除规则，否则合法地址可能被提前返回 404。

## 公开响应缓存

内置缓存只缓存符合条件的公开 GET/HEAD 响应：

- 只保存 HTTP 200 且 body 非空的响应。
- 带身份或个性化 Cookie 的请求不进入普通公开缓存。
- 带 `Set-Cookie` 的响应不能被当作公共页面复用。
- 缓存键包括方法、路径和查询参数。
- 条目达到上限时按到期情况淘汰。
- 内容及后台写操作会使当前进程缓存失效。

默认 TTL 为 30 秒、最大条目 512。缓存、计数和索引都在单进程内存中，多实例不会共享。

## 分类限流

WAF 区分动态、普通静态和上传附件资源：

| 类别 | 典型路径 | 默认窗口/次数 |
|---|---|---|
| 动态请求 | 普通页面和应用请求 | 60 秒 / 300 次 |
| 静态资源 | `/admin/assets/`、主题静态资源 | 60 秒 / 1200 次 |
| 上传资源 | `/uploads/` | 60 秒 / 600 次 |
| 搜索 | 搜索请求 | 60 秒 / 20 次 |
| XML-RPC | XML-RPC 请求 | 60 秒 / 30 次 |

每项均有独立开关、窗口和次数。分类频率计数按客户端 IP和类别存储。限制设置得过低会影响正常页面加载，因为一个页面通常同时请求多个静态资源。

## 附件下载封禁

附件下载封禁与普通上传资源限流独立。它按时间窗口内的附件请求次数统计，不按文件大小：

- 默认 60 秒内允许 120 次。
- 达到阈值后默认封禁 600 秒。
- 适用于防止反复请求大图片或下载文件消耗带宽。

按次数无法精确反映 Range 请求和真实字节量，反向代理/CDN 层仍应配置流量和并发限制。若附件通过对象存储域名直出，请在对象存储或 CDN 上配置对应策略，应用层 WAF看不到这些请求。

## 无效路径封禁

公开 URL 索引判定路径不存在时返回 404，并按 IP 统计：

- 默认窗口 60 秒。
- 默认阈值 20 次。
- 默认封禁 600 秒。

此策略主要针对随机 URL CC。插件新增路由会进入 `pluginRouteMayHandle` 判断，因此正常登记且已启用的插件路由不会被索引当成非法路径。

## 登录爆破防护

登录失败使用独立计数和封禁集合：

- 默认 300 秒窗口内允许 5 次失败。
- 触发后默认封禁登录 900 秒。
- 与普通请求封禁分开保存。

登录成功及具体计数清理按处理器逻辑执行。多实例部署时攻击者可轮换实例，因此还应在负载均衡器层进行统一限制。

## 反向代理 IP 信任

客户端 IP 默认来自 TCP 连接的 `RemoteAddr`。开启转发头信任后，只有直接连接的代理地址符合规则时，才按顺序读取 `X-Forwarded-For` 的第一个 IP或 `X-Real-IP`。

两种模式：

- `allowlist`：直接连接 IP 匹配地址组时才信任头。生产环境推荐。
- `denylist`：直接连接 IP 不匹配地址组时信任头，适合受控网络但边界更宽。

地址组一行一个 IPv4、IPv6 或 CIDR，空行和 `#` 开头的行忽略：

```text
127.0.0.1
10.0.0.0/8
2001:db8:1234::/48
```

若 GopherInk 可被公网直接访问，denylist 或过宽 CIDR 会允许伪造转发头。应在防火墙层只允许代理连接，并使用 allowlist 填写真实代理出口地址。

## 安全响应头和 HSTS

响应 writer统一设置：

- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `X-XSS-Protection: 0`

HSTS 仅在请求为 TLS 且开关启用时发送：

```text
Strict-Transport-Security: max-age=31536000; includeSubDomains
```

只有主域和所有子域都长期稳定支持 HTTPS 时才启用。浏览器会在有效期内拒绝降级 HTTP，证书过期或子域不支持 HTTPS 时可能造成不可访问。

## WAF 日志

事件独立写入：

```text
data/waf.log
```

`GOPHERINK_DATA_DIR` 会改变其父目录。后台 WAF 的“日志”选项卡可滚动查看和清空。最大记录条目默认 1000，可配置 1 到 100000；写入后超过上限会裁剪旧记录。

日志可能包含客户端 IP和请求路径，应限制文件访问权限并纳入隐私和保留策略。不要通过前台静态目录暴露 `data/`。

## 状态容量

WAF 对缓存、索引和计数状态设置最大条目，避免攻击者用不同 IP/路径无限增长内存。默认状态上限 100000，索引上限 10000。设置过低可能频繁丢弃正常状态，设置过高会增加攻击期间内存占用。

## SSRF 和外发请求

Pingback、Trackback发现等外发 HTTP 使用 `pkg/httpclient`：

- 只允许 HTTP/HTTPS。
- DNS 解析后拒绝回环、私网、未指定、多播和链路本地地址。
- 最多跟随 5 次重定向，并对重定向目标重复校验。
- 响应体有大小上限，默认 1 MB，并提供有限重试。

插件如果自行使用 `http.DefaultClient`，不会自动继承这些保护。访问用户提供 URL 时应复用安全客户端或实现等价校验。
